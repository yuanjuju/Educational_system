#### 1.1 增加功能

##### 1.1.1 增加学生

###### 基本信息

**请求路径：**`/admin/add_student`

**请求方式：**`POST`

###### 请求参数

参数格式：queryString

参数说明：

| 参数名          | 类型   | 是否必填 | 说明           |
| --------------- | ------ | -------- | -------------- |
| student_no      | string | 是       | 学生学号       |
| student_name    | string | 是       | 学生姓名       |
| student_sex     | string | 否       | 性别（不必须） |
| student_dept_id | string | 否       | 所在系别id     |
| student_DOB     | string | 否       | 性别           |
| Username        | string | 是       | 用户名         |
| password        | string | 是       | 密码           |

> student_id使用系统递增
>
> student_password  string  是  学生密码，无需请求，默认和学号一致
>
> 如果添加通过邮箱找回密码的功能，则添加上student_email

###### 响应数据

参数格式：application/json

参数说明：

| 参数名  | 类型   | 是否必填 | 说明                             |
| ------- | ------ | -------- | -------------------------------- |
| code    | int    | 是       | 状态码，200表示成功，400表示失败 |
| message | string | 是       | 响应消息                         |

**示例响应（成功）**:

```json
{
    "code": 200,
    "message": "学生添加成功"
}
```

**示例响应（失败）**:

```json
{
    "code": 400,
    "message": "学生添加失败"
}
```

#### 1.4 查找功能（多条件组合查询，使用关键字like，支持分页显示）

> 进入查找页面，首先默认是全部的信息列表**（请求参数只有page和pageSize，响应数据和下述系统）**。
>
> - 若未输入任何查询条件就点击查询按钮会弹出提示弹窗
> - 若无符合输入条件的信息，则返回空列表并提示没有符合条件的信息
> - 否则返回符合条件的信息列表

##### 1.4.1 查找学生

###### 基本信息

**请求路径：**`/admin/find_student?student_no={student_no}&student_name={student_name}&student_dept={student_dept}&student_DOB={student_DOB}&student_sex={student_sex}&page={page}&pageSize={pageSize}`

**请求方式：**`GET`

###### 请求参数

参数格式：queryString

参数说明：（以下参数可以是关键字，比如只输一个“王”可以查找到“王五”）

| 参数名       | 数据类型         | 是否必填 | 说明                                       |
| ------------ | ---------------- | -------- | ------------------------------------------ |
| student_no   | string           | 否       | 学号                                       |
| student_name | string           | 否       | 姓名                                       |
| student_dept | string           | 否       | 院系名称                                   |
| student_DOB  | string           | 否       | 生日                                       |
| student_sex  | enum("男"，"女") | 否       | 性别                                       |
| page         | int              | 是       | 分页查询的页码，如果未指定，默认为1        |
| pageSize     | int              | 是       | 分页查询的每页记录数，如果未指定，默认为20 |

###### 响应数据

参数格式：application/json

参数说明：

| 参数名  | 类型   | 是否必填 | 说明                             |
| ------- | ------ | -------- | -------------------------------- |
| code    | int    | 是       | 状态码，200表示成功，400表示失败 |
| message | string | 是       | 响应消息                         |
| total   | int    | 是       | 总记录条数                       |
| data    | array  | 否       | 学生列表（成功时返回）           |

**示例响应（成功）**:

```json
{
    "code": 200,
    "message": "查找学生成功",
    "total": 2,
    "data": [
        {
            "student_id": 1001,  //响应，但不显示在页面中，用作修改和删除的参数
            "student_no": "22020007001",
            "student_name": "张三",
            "student_dept": "信息科学与工程学部",
            "student_DOB": "2002-04-15",
            "student_sex": "男"
        },
        {
            "student_id": 1002,
            "student_no": "22020007002",
            "student_name": "李四",
            "student_dept": "工程学院",
            "student_DOB": "2003-04-15",
            "student_sex": "男"
        }
    ]
}
```

**示例响应（失败）**:

```json
{
    "code": 400,
    "message": "查找学生失败",
    "data":null
}
```

#### 4.1 登录

##### 4.1.1 基本信息

**请求路径：**`/login`

**请求方式：**`POST`

用户登录接口，支持管理员、教师、学生角色。登录完毕后，系统下发JWT令牌。 

##### 4.1.2 请求参数

参数格式：queryString

参数说明：

| 名称     | 类型   | 是否必须 | 备注                                       |
| -------- | ------ | -------- | ------------------------------------------ |
| id       | string | 是       | 学工号（student_id或teacher_id或admin_id） |
| password | string | 是       | 密码（加密后传输）                         |
| role     | string | 是       | 用户角色（admin、teacher、student）        |

请求数据样例：

```json
{
	"username": "jinyong",
    "password": "123456",
    "role": "admin" 
}
```

##### 4.1.3 响应数据

参数格式：application/json

参数说明：

| 名称 | 类型   | 是否必须 | 备注                             |
| ---- | ------ | -------- | -------------------------------- |
| code | int    | 是       | 状态码，200表示成功，400表示失败 |
| msg  | string | 否       | 提示信息                         |
| data | string | 是       | 返回的数据 , jwt令牌             |

响应数据样例：

```json
{
  "code": 200,
  "msg": "success",
  "data": "eyJhbGciOiJIUzI1NiJ9.eyJuYW1lIjoi6YeR5bq4IiwiaWQiOjEsInVzZXJuYW1lIjoiamlueW9uZyIsImV4cCI6MTY2MjIwNzA0OH0.KkUc_CXJZJ8Dd063eImx4H9Ojfrr6XMJ-yVzaWCVZCo"
}
```

##### 4.1.4 备注说明

> 用户登录成功后，系统会自动下发JWT令牌，然后在后续的每次请求中，都需要在请求头header中携带到服务端，请求头的名称为 token ，值为 登录时下发的JWT令牌。
>
> 如果检测到用户未登录，则会返回如下固定错误信息：
>
> ```json
> {
> 	"code": 400,
> 	"msg": "NOT_LOGIN",
> 	"data": null
> }
> ```
